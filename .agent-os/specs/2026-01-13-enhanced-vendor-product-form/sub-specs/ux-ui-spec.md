# UX/UI Specification: Enhanced Vendor Product Form

## Design Philosophy

Transform the product form from a simple 3-field sheet to a comprehensive yet approachable product editor. Use progressive disclosure to manage complexity while enabling power users to access all features.

## Layout Strategy

### Container: Full-Page vs Sheet

**Decision**: Transition from Sheet (slide-out panel) to Full Page for enhanced form.

**Rationale**:
- Sheet is suitable for 3 fields but cramped for 25+
- Full page provides breathing room for complex sections
- Better mobile experience
- Allows side-by-side preview (future enhancement)

**Implementation**:
- Route: `/dashboard/products/new` and `/dashboard/products/[id]/edit`
- Keep Sheet for quick edits (name/description only)
- "Full Editor" link from Sheet opens full page

## Interaction Patterns

### Adding Array Items

1. Click "+ Add" button
2. New row appears with focus on first field
3. Empty rows are not saved (filtered on submit)

### Removing Array Items

1. Click trash icon
2. Toast appears: "Specification removed" with [Undo] button
3. Undo restores the item
4. After 5 seconds, toast dismisses and removal is permanent

### Validation Feedback

- Real-time validation on blur
- Red border and error message below field
- Error count badge on collapsed sections with errors
- "Fix X errors to save" message near submit button

## Mobile Considerations

### Responsive Breakpoints

- **Desktop (â‰¥1024px)**: Full layout with side-by-side fields where appropriate
- **Tablet (768-1023px)**: Single column, sections remain collapsible
- **Mobile (<768px)**: Full-width fields, simplified interactions

### Mobile-Specific Adaptations

1. **Image management**: Simplified list view, tap to edit
2. **Specifications**: Full-width rows, swipe to delete
3. **Multi-select**: Bottom sheet instead of dropdown
4. **Icon selector**: Grid view in bottom sheet

## Accessibility Requirements

### Keyboard Navigation

| Key | Action |
|-----|--------|
| Tab | Move between form fields |
| Enter | Submit form / Expand section |
| Space | Toggle checkbox / Select option |
| Escape | Close dropdowns / Cancel |
| Arrow Up/Down | Navigate dropdown options |

### Screen Reader Support

- All sections have descriptive headings (h2, h3)
- Form fields have associated labels
- Error messages are announced via `aria-live`
- Array operations announce: "Image added", "Specification removed"

### Focus Indicators

- Visible focus ring on all interactive elements
- Focus trapped within modal dialogs
- Focus returned to trigger after modal close
